Loading Window Research

Best practices for a desktop loading experience in SharkReader.

TL;DR

Use an immediate app shell with skeletons, not a decorative full-screen splash. Keep startup non-blocking and reserve full overlays for real long-running tasks.

Key findings

  • Electron startup feels faster when the real window appears immediately with the correct background color.
  • Skeletons communicate progress better than a separate startup card.
  • Status text should describe real work, not generic loading copy.
  • A startup overlay should never be the single point of failure for app readiness.

Recommendation for SharkReader

Show topbar, sidebar, and a skeleton library grid from the first frame. Keep one quiet status line like Cargando biblioteca. Only use full-screen progress overlays for imports, repairs, and migrations.
